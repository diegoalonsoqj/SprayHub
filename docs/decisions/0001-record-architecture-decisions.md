# 1. Record architecture decisions

- Status: Accepted
- Date: 2026-06-29

## Context

We need to record the architectural decisions made on this project to give
future contributors the context behind important choices.

## Decision

We will use Architecture Decision Records, as described by Michael Nygard, in
the [`docs/decisions`](.) directory. Each record is a short Markdown file named
`NNNN-title-with-dashes.md`.

## Consequences

- Decisions are documented, reviewable, and versioned alongside the code.
- New contributors can understand *why*, not just *what*.
